## 3. Session 1 — the launch script

1. Open **Claude Code Desktop** in the Holmes working folder.
2. Paste `holmes-master-build-loop-v2.md` whole. It is the prompt; nothing precedes or wraps it.
3. The loop runs **pre-flight**: fetch origin, verify staged files, reconcile git state. A shortened `main` history is a **stop-and-escalate** (Gate-6-class), never something to merge onto.
4. The loop stops **BLOCKED at Gate Zero** and asks the four calls. Answer template — four one-liners:
   - `1. Audit accepted as charter verdict: YES`
   - `2. D-01 ratified: AGPL-3.0-or-later. Alfred/LICENSE reads: "<paste the quoted line(s)>"`
   - `3. Branch reconciliation (if raised): <APPROVED / decision>`
   - `4. Loop discipline confirmed: YES`
5. The loop executes **Task 0.3 first** (the deadline overrides ordering), then the remaining Phase 0 tasks — on a **session branch**. `main` only ever moves by **PR on explicit go-ahead** (Rule 9). The GitHub PR diff is the review surface.
6. At each Rule 9 checkpoint the loop emits a **readout** with VERIFIED/INFERRED/ASSUMED/UNKNOWN labels → paste it into this claude.ai project → pressure-test against spec, constitution, and AC doc → corrections back → explicit go-ahead → PR merges → **re-sync the connector** → next task.

2. Denylist enforced in **compiled Rust** at every layer, with adversarial regression tests passing — declared ≠ enforced is the standing standard.
3. The four perimeter obligations **verified in Alfred** (not [DIRECTIONAL]); `holmes-guard` adopted; Alfred's TS guard retired.

Honest limits carried from the loop, stated at ship exactly as during build: the denylist is **not fork-proof**, and misuse under mass distribution is **not structurally contained** — never claim otherwise in release material.
